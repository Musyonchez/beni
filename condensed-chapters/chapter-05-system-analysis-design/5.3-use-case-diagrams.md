# 5.3 Use Case Diagrams

Use case diagrams visualize system functionality from users' perspectives, identifying actors (entities interacting with the system) and use cases (discrete functional requirements). The system serves multiple distinct actor types with specific interaction patterns.

## Actors Identification

1. **Student (Customer)** - Primary end-user who browses menus, places orders, makes payments, tracks deliveries, participates in loyalty program
2. **Cafeteria Staff** - Employees who manage daily operations including order queue monitoring, status updates, inventory management
3. **System Administrator** - Personnel with comprehensive access for menu management, user administration, analytics, system configuration
4. **Payment Gateway (M-Pesa)** - External system processing financial transactions
5. **Firebase Authentication Service** - External system providing identity verification  
6. **Firebase Cloud Messaging** - External service delivering push notifications

## Primary Student Use Cases

- **UC1: Register Account** - Create account providing name, email, phone, password. Validate formats, send verification email.
- **UC2: Browse Menu and Place Order** - View categorized menu with images, descriptions, prices. Search, filter by category, view nutritional info. Add to cart with quantities.
- **UC3: Checkout and Payment** - Review cart, specify pickup time, enter special instructions, select payment. Initiate M-Pesa STK Push. Create order upon payment.
- **UC4: Track Order Status** - Monitor preparation progress (Order Received â†’ Payment Confirmed â†’ Preparing â†’ Ready â†’ Completed). Receive push notifications.
- **UC5: Manage Loyalty Points** - View points balance, earn points (1 per 100 KES), review transaction history, redeem points for discounts (100 points = 50 KES).

## Primary Staff Use Cases

- **UC6: Manage Order Queue** - View all pending orders chronologically or by priority. Display customer name, items, quantities, special instructions, payment status, elapsed time.
- **UC7: Update Order Status** - Transition orders through preparation workflow with one-tap updates. Changing to "Preparing" moves to active list. Marking "Ready" triggers push notification. Completing archives and triggers loyalty calculation.
- **UC8: Manage Inventory Availability** - Mark items unavailable when ingredients run out. Items display greyed out in student app. Re-enable when inventory replenishes.
