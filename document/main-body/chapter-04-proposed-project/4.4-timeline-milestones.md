# 4.4 Timeline and Milestones

## Week-by-Week Project Timeline

| Week | Phase | Key Activities | Deliverables | Milestone |
|------|-------|----------------|--------------|-----------|
| **1** | Research & Design | Literature review, problem analysis, requirements gathering | Chapters 1-2 drafted, requirements doc | ✓ Problem defined |
| **2** | Research & Design | System architecture design, database schema, wireframes | Architecture diagram, Firestore schema, UI wireframes | ✓ Design complete |
| **3** | Research & Design | Technology setup, Chapter 3 writing | Dev environment configured, objectives defined | ✓ Ready to code |
| **4** | Development | Firebase setup, authentication, basic UI structure | Login/register functional, app frame | ✓ Foundation built |
| **5** | Development | Menu browsing, cart management | Menu display, add-to-cart working | ✓ Core UI done |
| **6** | Development | Order placement, Firestore integration | Order submission functional | ✓ Ordering works |
| **7** | Development | M-Pesa integration (Daraja API) | Payment in sandbox successful | ✓ Payments integrated |
| **8** | Development | Loyalty points, order tracking, notifications | Points system, real-time tracking | ✓ All features done |
| **9** | Testing & Delivery | Testing (functional, performance), admin dashboard | Test results, dashboard deployed | ✓ System tested |
| **10** | Testing & Delivery | User testing, bug fixes, documentation | User feedback, final report | ✓ Project complete |

## Critical Path Dependencies

1. **Authentication** (Week 4) → **Order Placement** (Week 6)  
   *Users must login before placing orders*

2. **Order Placement** (Week 6) → **M-Pesa Integration** (Week 7)  
   *Orders must exist before accepting payments*

3. **M-Pesa Integration** (Week 7) → **Loyalty Points** (Week 8)  
   *Payment completion triggers point earning*

4. **All Features** (Week 8) → **User Testing** (Week 9)  
   *Complete system required for meaningful user feedback*

## Milestone Checkpoints

**Week 3 Checkpoint (End of Phase 1)**  
- [ ] Requirements approved by supervisor
- [ ] Design documents completed
- [ ] Development environment tested
- **Decision Point**: Proceed to development or refine design?

**Week 6 Checkpoint (Mid-Development)**  
- [ ] Basic ordering workflow functional
- [ ] Firebase database working correctly
- [ ] Timeline on track (no major delays)
- **Decision Point**: Continue with payments or address fundamental issues?

**Week 8 Checkpoint (End of Phase 2)**  
- [ ] All core features implemented
- [ ] No critical bugs blocking testing
- [ ] Admin dashboard functional
- **Decision Point**: Proceed to testing or extend development 1 week?

**Week 10 Final Deadline**  
- [ ] All testing complete
- [ ] Project report submitted
- [ ] Application deployed/demo ready

## Time Allocation (150-200 hour total budget)

- **Research & Design**: 45-60 hours (30%)
- **Development**: 75-100 hours (50%)
- **Testing & Documentation**: 30-40 hours (20%)

## Contingency Planning

**If ahead of schedule (Week 6-7)**:
- Implement bonus features (ratings/reviews, favorites)
- Polish UI/UX with animations
- Add offline caching for better performance

**If behind schedule (Week 7-8)**:
- Reduce admin dashboard to minimal viable features
- Simplify loyalty points (direct discounts vs complex tiers)
- Focus on core ordering + M-Pesa for demo
