## 4.4.5.1 Context Diagram (Level 0)

The context diagram provides the highest-level view of the FarmLink system, showing it as a single process that interacts with all external entities. It does not show internal processes — only the data that flows in and out of the system boundary.

### External Entities and Data Flows

| Entity | Data Into System | Data Out of System |
|---|---|---|
| Farmer | Registration details, listing data, order status updates, profile updates | Account confirmation, listing confirmation, order notifications, earnings data |
| Buyer | Registration details, search queries, cart/order data, M-Pesa phone, review data | Account confirmation, search results, order confirmation, tracking status, payment result |
| Administrator | Login credentials, moderation actions, report requests | User lists, listing lists, sales reports, user reports |
| Safaricom (M-Pesa) | Payment callback (transaction result) | STK Push request (amount, phone, reference) |
| Twilio | SMS delivery status | SMS message content and recipient number |

## 4.4.5.2 Level 1 Diagram

The Level 1 DFD decomposes the FarmLink system into its six main sub-processes, showing the data stores and the flows between processes and entities.

**[INSERT LEVEL 1 DFD HERE]**

*Figure 4.6 — FarmLink Level 1 Data Flow Diagram*

### Sub-Processes

| Process | Description |
|---|---|
| P1 — User Management | Handles registration, login, profile updates, and admin user management |
| P2 — Product Management | Handles create, read, update, delete operations on produce listings and geolocation search |
| P3 — Order Management | Handles order placement, status transitions, and order history retrieval |
| P4 — Payment Processing | Handles M-Pesa STK Push initiation and Daraja API callback processing |
| P5 — Review Management | Handles review submission, farmer rating calculation, and review retrieval |
| P6 — Notification Service | Handles push notification and SMS dispatch on order events |

### Data Stores

| Store | Contents |
|---|---|
| D1 — Users | All registered farmer, buyer, and admin accounts |
| D2 — Products | All produce listings with geolocation data |
| D3 — Orders | All placed orders with status history and payment records |
| D4 — Reviews | All submitted ratings and comments |
| D5 — Notifications | All in-app notification records |

## 4.4.5.3 Level 2 Diagram

The Level 2 DFD further decomposes Process 3 (Order Management) — the most complex sub-process — into its constituent operations.

**[INSERT LEVEL 2 DFD HERE]**

*Figure 4.7 — FarmLink Level 2 DFD: Order Management*

### Order Management Sub-Processes

| Sub-Process | Input | Output |
|---|---|---|
| P3.1 — Place Order | Cart data from Buyer, Product data from D2 | Order record written to D3, Payment request to P4 |
| P3.2 — Confirm Payment | Payment result from P4 | Order paymentStatus updated in D3, Notification to Farmer via P6 |
| P3.3 — Update Order Status | Status update from Farmer | Order status updated in D3, Notification to Buyer via P6 |
| P3.4 — Cancel Order | Cancel request from Buyer | Order status set to cancelled in D3, Notification to Farmer via P6 |
| P3.5 — Retrieve Order | Order ID from Buyer or Farmer | Order data read from D3, returned to requester |
| P3.6 — Trigger Review | Delivered status from P3.3 | Review prompt triggered to Buyer, Review record written to D4 |
