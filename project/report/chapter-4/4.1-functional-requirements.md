# 4.1 System Requirements

## 4.1.1 Functional Requirements

Functional requirements define the specific behaviours and operations that the FarmLink system must be capable of performing. The following functional requirements were identified through user interviews, questionnaire analysis, and review of comparable systems.

| Req. ID | Requirement | User Role |
|---|---|---|
| FR01 | The system shall allow new users to register with a name, email, password, phone number, and role (farmer or buyer) | All |
| FR02 | The system shall authenticate registered users via email and password and issue a secure JWT session token | All |
| FR03 | The system shall allow farmers to create produce listings with a title, category, price, quantity, unit, images, and GPS location | Farmer |
| FR04 | The system shall allow farmers to edit and delete their own listings | Farmer |
| FR05 | The system shall allow buyers to browse all available produce listings filtered by category | Buyer |
| FR06 | The system shall allow buyers to search for produce and farmers within a specified radius of their current GPS location | Buyer |
| FR07 | The system shall allow buyers to view a detailed farmer profile including bio, location, average rating, and all active listings | Buyer |
| FR08 | The system shall allow buyers to add produce items to a shopping cart and adjust quantities | Buyer |
| FR09 | The system shall allow buyers to place an order and initiate payment via M-Pesa STK Push | Buyer |
| FR10 | The system shall update order payment status upon receiving a confirmed callback from the Safaricom Daraja API | System |
| FR11 | The system shall allow buyers to view the real-time status of their orders | Buyer |
| FR12 | The system shall allow farmers to update the status of received orders (confirmed, dispatched, delivered) | Farmer |
| FR13 | The system shall send push notifications and SMS alerts to the relevant user at each order status change | System |
| FR14 | The system shall allow buyers to submit a star rating and written review for a farmer upon order delivery | Buyer |
| FR15 | The system shall calculate and display a farmer's average rating based on all received reviews | System |
| FR16 | The system shall allow administrators to view, verify, and deactivate user accounts | Admin |
| FR17 | The system shall allow administrators to remove produce listings that violate platform guidelines | Admin |
| FR18 | The system shall generate sales and user registration reports for administrators | Admin |
