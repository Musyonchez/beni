# 4.4.1 Systems Architecture

FarmLink follows a three-tier client-server architecture comprising a mobile presentation layer, a business logic layer, and a data layer. The three tiers are fully decoupled and communicate exclusively through a RESTful HTTP API.

## Description of Architecture Layers

**Tier 1 — Presentation Layer (Mobile Client)**
The presentation layer is a web application built using Next.js 14 with the App Router. It runs in any modern web browser on desktop and mobile devices and is responsible for rendering all user interface screens, handling user input, managing local session state via browser localStorage, and communicating with the backend API via Axios HTTP calls. The presentation layer contains no business logic — all data processing is delegated to the backend.

**Tier 2 — Business Logic Layer (Backend Server)**
The backend is a Node.js application built with the Express.js framework, running on port 5000. It exposes a set of RESTful API endpoints organised into six domain modules: authentication, products, orders, payments, reviews, and notifications. The backend is responsible for enforcing all business rules, validating incoming requests, managing JWT-based authentication and role-based access control, and orchestrating calls to external services. Middleware components handle cross-cutting concerns including CORS policy, request validation, and centralised error handling.

**Tier 3 — Data Layer (Database)**
The data layer consists of a MongoDB database hosted on MongoDB Atlas. Mongoose is used as the Object Document Mapper (ODM) to define collection schemas and perform database queries from the backend. The database stores five primary collections: users, products, orders, reviews, and notifications. Geospatial indexes on the users and products collections enable efficient proximity-based queries for the geolocation feature.

**External Services**
The architecture integrates two external services: the Safaricom Daraja API for M-Pesa payment processing, and Twilio for SMS notifications dispatched to farmers and buyers at each order status change. All external service calls are made exclusively from the backend to protect API credentials from exposure in the web client.
