# 4.4.3 Flowchart

The flowchart below illustrates the primary buyer purchase flow — from opening the FarmLink application through to leaving a review after delivery. This represents the core end-to-end user journey that the system is designed to support.

**[INSERT FLOWCHART DIAGRAM HERE]**

*Figure 4.3 — FarmLink Buyer Purchase Flowchart*

## Flowchart Steps

```
START
  │
  ▼
Open FarmLink App
  │
  ▼
User Registered? ──No──► Register (name, email, password, phone, role=buyer)
  │                              │
  Yes                            ▼
  │                       Account Created
  │◄──────────────────────────────┘
  ▼
Login (email + password)
  │
  ▼
Credentials Valid? ──No──► Show Error → Return to Login
  │
  Yes
  ▼
Home Feed Loaded (nearby produce, categories)
  │
  ▼
Browse / Search Produce
  │
  ├── By Category → Filter listings by category
  └── By Location → Enable GPS → Show nearby farmers on map
  │
  ▼
Select Product → View Product Detail Page
  │
  ▼
Add to Cart
  │
  ▼
Continue Shopping? ──Yes──► Return to Browse
  │
  No
  ▼
View Cart (items, quantities, total in KES)
  │
  ▼
Proceed to Checkout
  │
  ▼
Confirm M-Pesa Phone Number
  │
  ▼
Tap "Pay via M-Pesa"
  │
  ▼
STK Push Sent to Buyer's Phone
  │
  ▼
Buyer Enters M-Pesa PIN on Phone
  │
  ▼
Payment Successful? ──No──► Show Payment Failed → Option to Retry
  │
  Yes
  ▼
Order Placed → Farmer Notified (push + SMS)
  │
  ▼
Order Tracking Screen (status: Pending)
  │
  ▼
Farmer Confirms Order → Status: Confirmed → Buyer Notified
  │
  ▼
Farmer Dispatches → Status: Dispatched → Buyer Notified
  │
  ▼
Order Delivered → Status: Delivered → Buyer Notified
  │
  ▼
Review Prompt Shown
  │
  ▼
Buyer Leaves Rating (1–5 stars) + Comment
  │
  ▼
Review Submitted → Farmer Average Rating Updated
  │
  ▼
END
```
