# 6.4 Future Work

The current FarmLink prototype demonstrates the core functionality required for a viable agricultural eMarketplace in the Kenyan context. However, the 14-week project timeline and the constraints of a student prototype necessarily limit the scope of features that could be implemented. The following enhancements are proposed for future development iterations of the platform.

**Swahili Language Support.** The current FarmLink interface is implemented in English. Given that a significant proportion of smallholder farmers in the target region are more comfortable communicating in Swahili, adding full Swahili localisation would substantially improve accessibility and reduce the literacy barrier for first-time users. A bilingual English-Swahili interface should be a priority in the next development cycle.

**Farmer Cooperative Group Accounts.** The current system supports individual farmer accounts. Future versions should introduce a group account model that allows registered cooperatives and SACCOs to maintain a shared storefront, assign roles to member farmers, and distribute earnings to individual members through M-Pesa disbursements. This would align the platform with the cooperative farming structures that are prevalent in Kiambu County and would improve the volume and diversity of produce available to buyers.

**Third-Party Delivery Coordination.** The current system does not manage physical logistics. A future integration with motorcycle courier APIs — or a built-in rider assignment module — would allow farmers and buyers to coordinate last-mile delivery directly through the application, removing a key friction point in the purchase flow and making the platform viable for buyers who cannot travel to the farm for pickup.

**Crop Price Forecasting.** As transaction data accumulates on the platform, a data analytics module could be introduced to provide farmers with historical price trends, seasonal demand patterns, and suggested pricing for specific produce categories. This feature would directly address the price opacity challenge identified in the research findings and give farmers the market intelligence they need to make informed planting and selling decisions.

**Dedicated Admin Dashboard.** The current administrator interface shares the same Next.js web application as the farmer and buyer interfaces, distinguished only by role-based access. A dedicated standalone admin dashboard — with richer data visualisation, bulk action tools, and exportable reports — would provide the system administrator with a more comprehensive view of platform activity than the current shared interface allows.
